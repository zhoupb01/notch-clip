# 01 · 产品需求文档（PRD）

## 1. 背景与定位

macOS 自带剪贴板只保留最后一条记录，复制新内容会覆盖旧内容。剪贴板历史工具（Maccy、Paste 等）解决了这个问题，但形态上都是「菜单栏图标 + 弹窗」。本产品把入口换成 MacBook 的刘海区域，理由：

- 刘海位于屏幕顶部正中央，是视觉焦点，也是鼠标最容易「甩」到的位置；
- 刘海本身是一块无法利用的黑色区域，把 UI 藏在里面是零成本的（平时不占任何屏幕空间）；
- 复制动作高频且瞬时，用户需要的是「余光可见的反馈」而不是「切换窗口去确认」，刘海展开式的 HUD 恰好满足这一点（类比 iPhone 灵动岛）。

产品原则：**平时隐形、复制有反馈、呼出即粘贴、数据全本地。**

## 2. 目标用户

- 主力人群：开发者、设计师、文字工作者——每天复制粘贴几十到几百次的人。
- 设备：带刘海的 MacBook 为主；无刘海设备（外接屏、旧机型）通过降级形态兼容。

## 3. 功能需求

优先级定义：P0 = v1 必须有；P1 = v1 尽量有（在 M6 里程碑实现）；P2 = 本次不实现，仅保留设计空间。

### P0（核心闭环）

| 编号 | 功能 | 说明 |
|---|---|---|
| F1 | 剪贴板监听 | 后台轮询系统剪贴板，捕获新内容 |
| F2 | 历史存储 | 默认保留最近 500 条，本地持久化，重启不丢 |
| F3 | 类型识别 | 区分：纯文本 / 链接 / 颜色值 / 图片 / 文件，图标区分显示 |
| F4 | 三态交互 | 常态（隐形）/ HUD 反馈态 / 面板态，详见 03 文档 |
| F5 | 复制反馈 HUD | 检测到新内容时刘海展开约 1.2 秒，显示类型 + 摘要 |
| F6 | 历史面板 | 悬停刘海或 ⌘⇧V 呼出；列表 + 即输即搜 |
| F7 | 粘贴 | 回车粘贴选中项；⌘1~⌘9 直接粘贴对应序号项；鼠标点击粘贴 |
| F8 | 隐私过滤 | 跳过带 ConcealedType / TransientType 标记的内容（密码管理器等） |
| F9 | 焦点保持 | 全程不抢占目标应用焦点，粘贴落在用户原来的应用里 |
| F10 | 无刘海降级 | 无刘海屏幕上显示为顶部居中的小悬浮条，交互一致 |

### P1（打磨项，M6 实现）

| 编号 | 功能 | 说明 |
|---|---|---|
| F11 | 固定（Pin） | 常用片段固定在列表顶部，不被条数上限淘汰 |
| F12 | 应用排除列表 | 用户可按 Bundle ID 排除应用（如终端、密码管理器） |
| F13 | 状态栏菜单 | 一个 NSStatusItem，提供：打开面板 / 清空历史 / 设置 / 退出 |
| F14 | 开机自启 | 设置项，用 SMAppService 实现 |
| F15 | 清空历史 | 一键清空（保留固定项），需二次确认 |
| F16 | 历史上限设置 | 100 / 500 / 1000 三档可选 |

### P2（本次不实现）

- 拖拽暂存 shelf（把文件拖进刘海暂存，NotchDrop 的核心玩法）
- 顺序粘贴（连续复制多条后按顺序逐条粘贴）
- 富文本 / RTF 格式保留（v1 一律降级为纯文本）
- 快捷键自定义（v1 写死 ⌘⇧V）

### 明确不做

- 任何形式的云同步 / 账号系统 —— 数据全本地是卖点
- iOS 版本
- App Store 上架（v1 走直接分发，不开沙盒，见 02 文档决策 D6）

## 4. 非功能需求

| 指标 | 要求 |
|---|---|
| 空闲 CPU | < 1%（轮询定时器为唯一后台活动） |
| 内存 | 常驻 < 100 MB（图片只在面板打开时按需加载缩略图） |
| 启动时间 | 冷启动到监听就绪 < 1 秒 |
| 数据安全 | 所有数据存放在 `~/Library/Application Support/NotchClip/`，不发出任何网络请求 |
| 权限 | 仅申请「辅助功能」权限（用于模拟 ⌘V），首次粘贴时引导授权 |

## 5. 验收口径（产品层面）

1. 在任意应用中复制文本 → 1 秒内刘海出现 HUD 反馈并自动收回。
2. 在备忘录中定位光标 → 按 ⌘⇧V → 输入关键字 → 回车 → 内容粘贴进备忘录，且备忘录全程保持前台。
3. 从 1Password/浏览器密码框复制密码 → 历史中**不出现**该条目。
4. 重启应用 → 历史仍在。
5. 在无刘海的外接显示器为主屏时 → 顶部出现居中悬浮条，功能一致。
