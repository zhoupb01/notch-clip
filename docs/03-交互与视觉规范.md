# 03 · 交互与视觉规范

本文档所有尺寸单位为 pt。面板和 HUD 的配色**固定为暗色**（不跟随系统外观），因为它们视觉上是刘海的延伸，而刘海永远是黑的。

## 1. 三态总览

| 状态 | 触发进入 | 触发退出 | 鼠标事件 |
|---|---|---|---|
| idle 常态 | 默认 / 其他态退出后 | — | 穿透（ignoresMouseEvents = true） |
| hud 反馈态 | 监听到新剪贴板内容 | 1.2 秒后自动退出；期间再来新内容则刷新内容并重新计时 | 穿透 |
| panel 面板态 | ① 鼠标悬停刘海矩形 ≥ 0.15 秒；② ⌘⇧V | ① esc；② 点击面板外任意处；③ 完成一次粘贴；④ 鼠标移出面板区域外 ≥ 0.4 秒（仅悬停呼出时适用） | 接收 |

状态优先级：panel > hud。面板打开期间来了新复制内容 → 不显示 HUD，直接在列表顶部插入新行。

## 2. 常态（IdleNotchView）

- 一个纯黑（#000000）圆角矩形，尺寸与位置**精确等于** NotchGeometry 给出的刘海矩形，底部两角圆角 10。
- 有真刘海时它与硬件刘海重叠，用户不可见；作用是提供悬停检测的视觉锚点与 HUD 展开的起始形状。
- 无刘海降级形态见第 6 节。

## 3. HUD 反馈态（HUDView）

**几何**：以刘海为中心向左右展开。宽 = 刘海宽 + 160（左右各加 80），高 = 刘海高 + 8，底部圆角 16。始终水平居中于刘海。

**动画**：展开用 spring（response 0.35, dampingFraction 0.75）；1.2 秒后收回用 easeIn 0.18 秒。SwiftUI 对 frame 变化做动画即可。

**内容布局**（水平排列，左右内边距 16，元素间距 10）：

```
[类型图标 15pt] [主文本 12pt，单行截断，白色 92%] ……… [右侧标签 11pt，白色 45%]
```

各类型的内容规格：

| 类型 | 图标 (SF Symbols) | 主文本 | 右侧标签 |
|---|---|---|---|
| text | `doc.on.doc` | 内容前 40 字符 | `已复制 · 文本` |
| link | `link` | 去掉 scheme 的 URL | `已复制 · 链接` |
| color | 一个 14×14 圆角 4 的实心色块（用实际颜色填充） | 色值原文（等宽字体） | `已复制 · 颜色` |
| image | `photo` | `图片 宽×高` | `已复制 · 图片` |
| file | `doc` | 文件名（不含路径） | `已复制 · 文件` |

## 4. 面板态（PanelView）

**几何**：宽 420，水平居中于刘海；顶部与屏幕顶边平齐（从刘海「长」下来），底部圆角 18。高度自适应内容，最大 480。背景色 #0E0E10，不透明。展开/收起动画与 HUD 相同参数。

**结构（自上而下）**：

1. **顶部安全区**：高 = 刘海高，空白（避免内容被硬件刘海遮住）。
2. **搜索行**：高 34。左侧图标 `magnifyingglass` 13pt 白 45%；输入框占满剩余，placeholder 文案 `搜索剪贴板历史`，字号 12。⌘⇧V 呼出时自动聚焦；悬停呼出时不聚焦。下方 0.5pt 分隔线，白 12%。
3. **固定区**（有固定项时才显示）：固定项行，行首图标换成 `pin.fill`（琥珀色 #FAC775），其余同普通行。固定区下方一条分隔线。
4. **历史列表**：ScrollView，最多显示当前过滤结果的前 50 条（性能兜底）。
5. **底部提示条**：高 26，居中排布 11pt 白 35% 的提示文案：`↩ 粘贴    ⌘1-9 快速粘贴    esc 关闭`。上方 0.5pt 分隔线。

**列表行规格**（行高 36，左右内边距 12，元素间距 10）：

```
[类型图标/色块/缩略图 16pt] [主文本 12pt 白 90%，单行截断] [来源+时间 11pt 白 40%] [序号badge 11pt 白 35% 等宽]
```

- image 类型行首显示 24×24 圆角 4 的缩略图。
- 来源+时间格式：`Safari · 刚刚`；时间显示规则：< 1 分钟 `刚刚`，< 60 分钟 `N 分钟前`，< 24 小时 `N 小时前`，否则 `M月d日`。
- 序号 badge：列表可见顺序的前 9 条显示 `⌘1` ~ `⌘9`（固定项优先编号）。
- 选中态：整行背景白 9%，圆角 8。任意时刻有且只有一行处于选中态，初始为第一行。
- 悬停某行 → 该行成为选中行。

**键盘映射**（面板打开期间）：

| 按键 | 行为 |
|---|---|
| ↑ / ↓ | 移动选中行（到边界停住，不循环） |
| ↩ 回车 | 粘贴选中行 |
| ⌘1 ~ ⌘9 | 直接粘贴对应序号行 |
| esc | 关闭面板，不粘贴 |
| 其他可打印字符 | 进入搜索框（搜索框始终是文本焦点） |

**搜索规则**：对 `text` 字段做「不区分大小写的包含匹配」；image 条目在搜索非空时隐藏；输入即过滤，无防抖。

## 6. 边缘情况

| 情况 | 行为 |
|---|---|
| 无刘海屏幕 | 降级为「假刘海」：200×32 黑色圆角条，贴屏幕顶边水平居中。三态交互完全一致（这也是开发调试形态） |
| 多显示器 | 只在一块屏幕上显示：优先选 `safeAreaInsets.top > 0` 的内建屏，否则选主屏。监听 `didChangeScreenParametersNotification`，屏幕变化时重建窗口位置 |
| 全屏应用 | 因 `.fullScreenAuxiliary` + 高窗口层级，HUD 和面板照常显示在全屏内容之上 |
| 菜单栏自动隐藏 | 不做特殊处理，窗口位置只依赖屏幕 frame，与菜单栏无关 |
| 复制超长文本 | 存储不截断；HUD 与列表行只显示截断后的单行 |
| 剪贴板内容为空/不可解析 | 静默忽略，不产生条目、不显示 HUD |
| 辅助功能权限未授权 | 粘贴动作退化为「仅写回剪贴板 + 关面板」，同时弹一次引导弹窗（每次启动最多弹一次） |
