# NotchClip — macOS 刘海剪贴板历史工具

一句话定义：一个常驻在 MacBook 刘海位置的剪贴板历史管理器。平时完全隐形；复制时刘海短暂展开给出反馈；快捷键呼出历史面板，搜索并一键粘贴。

设计思路来源于三个开源/商业项目，取各自最成熟的部分：

| 借鉴对象 | 借鉴内容 |
|---|---|
| Maccy | 剪贴板内核：changeCount 轮询、隐私标记过滤、模拟 ⌘V 粘贴 |
| boring.notch | 刘海窗口壳：NSPanel 层级、刘海几何计算、展开/收起形态 |
| Vibe Island / NotchDrop | 产品形态：灵动岛式三态交互、无刘海降级方案 |

## 给实现模型的工作规则（必须遵守）

1. **严格按 04-实现指南.md 的里程碑顺序实现（M0 → M6），禁止跳跃、禁止提前实现后续里程碑的内容。**
2. 每个里程碑完成后：项目必须能编译通过，并逐条核对该里程碑的「验收清单」，输出一份简短的验收报告后再进入下一个里程碑。
3. **禁止引入任何第三方依赖**（不用 SPM 包、不用 CocoaPods）。本项目只用 AppKit、SwiftUI、Foundation、Carbon.HIToolbox、ServiceManagement 这些系统框架。
4. **禁止修改已锁死的架构决策**（见 02-架构设计.md 第 5 节）。如果你认为某个决策有问题，在验收报告中提出，但仍按文档实现。
5. 文档之间如有冲突，优先级为：**04-实现指南 > 03-交互规范 > 02-架构 > 01-PRD**。
6. 04 中给出的完整代码块可以直接使用（允许微调命名和排版），但不得改变其行为；给出「规格描述」而非代码的部分，由你按规格自行实现。
7. 遇到文档未覆盖的细节，选择**最简单、最不影响其他模块**的实现，并在验收报告中记录该决定。
8. 所有代码用 Swift 编写，注释用中文。不写单元测试（v1 用手动验收清单代替），但每个模块保持可单独调试。
9. 不要实现 PRD 中标记为 P2 或「明确不做」的功能，即使实现起来很容易。
10. UI 文案全部中文，参照 03 文档中给出的文案原文，不要自行发挥。

## 环境要求

- Xcode 15 或更新版本，macOS 14 (Sonoma) 或更新版本
- 最佳测试机：带刘海的 MacBook（2021 款及以后的 MacBook Pro / Air）
- 无刘海机器也可以开发调试：程序会自动降级为「顶部居中悬浮条」形态（见 03 文档第 6 节）
